# Автоматизация рабочих процессов дизайн-студии через AI и no-code инструменты

## Название проекта

**AI-Driven Design Ops: автоматизация рутины в студии «Пиксель»**

---

## Проблема

Студия из 4 дизайнеров и 2 верстальщиков тонула в рутине:

- На генерацию вариантов мудбордов уходило 2–3 дня
- Копирайтер не успевал за дизайнерами: тексты для лендингов писались 1–2 недели
- Ручная адаптация макетов под мобильные устройства занимала 20–30% времени проекта
- Клиенты забывали заполнять брифы, менеджер тратил 2 часа в день на напоминания
- SEO-описания писались «на потом» и часто не попадали в релиз
- Заявки с сайтов на Tilda терялись между формой и CRM

**Результат:** проект в 15 страниц растягивался на 6–8 недель вместо 3–4.

---

## Решение

Построил экосистему автоматизаций, где AI и no-code инструменты берут на себя рутинные задачи, а дизайнеры фокусируются на креативе и UX. Интеграция покрывает весь пайплайн: от брифа до передачи в разработку.

## Что автоматизировал

### 1. Генерация вариантов дизайна через Midjourney + Stable Diffusion

**Задача:** ускорить этап исследований и презентации концепций клиенту.

**Реализация:**

- Создал промпт-библиотеку из 50+ шаблонов для Midjourney, структурированных по типам проектов (SaaS, e-commerce, корпоративные сайты)
- Настроил автоматический upscale через Stable Diffusion API для выбранных вариантов
- Интегрировал генерацию с Notion: результаты падают в базу проекта с тегами по стилю и цветовой палитре

**Промпт-шаблон (пример):**

```
/imagine prompt: modern saas landing page hero section, 
minimalist interface, soft gradients in {primary_color} and 
{secondary_color}, floating 3d elements, clean typography, 
ultra-detailed, ui/ux design --ar 16:9 --v 6 --s 750
```

**Инструменты:** Midjourney API, Stable Diffusion (AUTOMATIC1111), Make.com, Notion API.

### 2. Автоматическая генерация текстов через GPT-4

**Задача:** избавить дизайнеров от «рыбы» и дать клиенту реалистичный контент уже на этапе прототипа.

**Реализация:**

- Настроил сценарий в n8n: после утверждения структуры сайта в Notion триггерится генерация текстов
- GPT-4 получает контекст: бриф клиента, ТЗ на раздел, ключевые слова, тональность (деловой / дружелюбный / технический)
- Генерация проходит в 2 этапа: сначала черновик, потом рерайт под заданные лимиты символов
- Результат записывается обратно в Notion и подставляется в Figma через плагин Notion → Figma

**Системный промпт для GPT-4:**

```
Ты — копирайтер для IT-компаний. Пиши тексты для лендингов 
в формате: заголовок H1 (до 60 символов), подзаголовок 
(до 120 символов), 3 преимущества с иконками (до 80 символов 
каждое). Тон: уверенный, но не агрессивный. Избегай канцеляризмов 
и жаргона. Целевая аудитория: руководители среднего бизнеса 
25–45 лет.
```

**Инструменты:** n8n, OpenAI API (GPT-4), Notion API, плагин Notion → Figma.

### 3. Вайбкодинг: плагины и скрипты для Figma

**Задача:** устранить повторяющиеся операции в Figma: переименование слоёв, генерацию спек, выгрузку ассетов.

**Реализация:**

#### Плагин «Auto Specs» (написан на TypeScript + Figma Plugin API)

Автоматически генерирует спецификацию для разработчика: отступы, цвета, шрифты, эффекты. Выделяешь фрейм → получаешь готовую спеку в виде таблицы рядом с макетом.

**Ключевой фрагмент кода:**

```typescript
// Псевдокод логики плагина
function generateSpecs(node: FrameNode) {
  const specs = {
    width: node.width,
    height: node.height,
    padding: {
      top: node.paddingTop,
      right: node.paddingRight,
      bottom: node.paddingBottom,
      left: node.paddingLeft
    },
    fills: node.fills.map(fill => 
      fill.type === 'SOLID' 
        ? rgbToHex(fill.color) 
        : fill.type
    ),
    typography: {
      fontFamily: node.fontName,
      fontSize: node.fontSize,
      lineHeight: node.lineHeight,
      letterSpacing: node.letterSpacing
    }
  };
  
  // Создание спецификационного фрейма
  const specFrame = figma.createFrame();
  specFrame.name = `${node.name} — specs`;
  specFrame.x = node.x + node.width + 40;
  specFrame.y = node.y;
  
  // Рендеринг текста спецификации
  const specText = figma.createText();
  specText.characters = formatSpecs(specs);
  specFrame.appendChild(specText);
}
```

#### Скрипт автопереименования слоёв (Python + FigJam REST API)

```python
import requests
import re

FIGMA_TOKEN = "figd_xxxxx"
FILE_KEY = "ABC123xyz"

def sanitize_layer_name(name: str) -> str:
    """Приводит имена слоёв к единому формату."""
    name = name.lower().strip()
    name = re.sub(r'[^a-z0-9_]', '_', name)
    name = re.sub(r'_+', '_', name)
    return name[:50]

def batch_rename_layers(nodes: list):
    """Пакетное переименование по схеме: тип_элемента / блок / описание."""
    for node in nodes:
        new_name = f"{node['type']}/{sanitize_layer_name(node['name'])}"
        # PATCH-запрос к Figma API для обновления имени
        requests.patch(
            f"https://api.figma.com/v1/files/{FILE_KEY}/nodes",
            headers={"X-Figma-Token": FIGMA_TOKEN},
            json={node['id']: {"name": new_name}}
        )
```

**Инструменты:** Figma Plugin API (TypeScript), Figma REST API, Python 3.11, Pyfigma.

### 4. Автоматическая генерация SEO-описаний

**Задача:** SEO не откладывалось на «потом», а генерировалось параллельно с дизайном.

**Реализация:**

- Сценарий в Make.com: при создании страницы в Notion (где ведётся структура сайта) автоматически генерируются meta-title и meta-description через GPT-3.5
- Учитываются: ключевые слова из брифа, конкуренты (парсятся через SERP API), длина в пикселях (чтобы не обрезалась в выдаче)
- Результат валидируется: длина title 30–60 символов, description 70–160 символов
- Готовые meta-теги экспортируются в CSV для передачи разработчикам

**Схема автоматизации (текстовое описание):**

```
[Notion: новая страница в базе «Структура сайта»]
    ↓ (Webhook)
[Make.com: сценарий «SEO-генерация»]
    ↓
[SERP API: анализ ТОП-10 выдачи по ключевому слову]
    ↓
[OpenAI GPT-3.5: генерация 3 вариантов title + description]
    ↓
[Фильтр: валидация длины]
    ↓
[Notion: запись в поля «Meta Title», «Meta Description»]
    ↓
[Google Sheets: экспорт в CSV-таблицу для dev-команды]
```

**Инструменты:** Make.com, SERP API, OpenAI API, Notion, Google Sheets.

### 5. Интеграция форм Tilda с CRM через webhook

**Задача:** заявки с лендингов на Tilda мгновенно попадают в CRM без потерь и дублей.

**Реализация:**

#### Схема интеграции (n8n)

```
[Tilda: отправка формы]
    ↓ (Webhook POST)
[n8n: эндпоинт /tilda-form]
    ↓
[Проверка: валидация email + телефона]
    ↓ (невалидно)
[Логирование ошибки → Telegram-бот менеджеру]
    ↓ (валидно)
[Дедупликация: проверка на существование в CRM]
    ↓ (дубль)
[Обновление существующей сделки]
    ↓ (новый лид)
[CRM (Битрикс24 / amoCRM): создание лида]
    ↓
[Telegram: уведомление менеджеру]
    ↓
[Google Sheets: архив всех заявок]
```

#### Пример payload от Tilda

```json
{
  "formid": "form123456",
  "formname": "Консультация дизайна",
  "Name": "Алексей Петров",
  "Email": "alex@company.ru",
  "Phone": "+7 (999) 123-45-67",
  "Comment": "Нужен редизайн корпоративного сайта",
  "utm_source": "google",
  "utm_medium": "cpc",
  "utm_campaign": "design_services_spring2024"
}
```

#### Обработка в n8n (логика узла Function)

```javascript
// Валидация и обогащение данных
const input = $input.first().json.body;

// Проверка телефона
const phoneRegex = /^\+7\s?\(?\d{3}\)?\s?\d{3}[-.\s]?\d{2}[-.\s]?\d{2}$/;
if (!phoneRegex.test(input.Phone)) {
  return [{ json: { error: "Invalid phone", raw: input }}];
}

// Форматирование телефона
const cleanPhone = input.Phone.replace(/\D/g, '');

// Определение источника
const source = input.utm_source 
  ? `Реклама: ${input.utm_source}` 
  : "Органический трафик";

return [{
  json: {
    name: input.Name,
    email: input.Email.toLowerCase(),
    phone: cleanPhone,
    comment: input.Comment,
    source: source,
    campaign: input.utm_campaign,
    created_at: new Date().toISOString()
  }
}];
```

**Инструменты:** n8n, Tilda Webhook, Битрикс24 REST API, Telegram Bot API, Google Sheets API.

### 6. Автоматическая адаптация макетов под разные разрешения

**Задача:** сократить время на ручную адаптацию desktop → tablet → mobile.

**Реализация:**

#### Плагин «Responsive» для Figma

Автоматически перестраивает макет под заданные брейкпоинты с сохранением визуальной иерархии.

**Принцип работы:**

1. Дизайнер задаёт правила адаптации для компонентов: «на 768px этот блок становится колонкой», «на 375px скрыть декоративные элементы»
2. Плагин клонирует desktop-фрейм и применяет правила
3. Автоматически пересчитывает отступы (8px grid), размеры шрифтов (scale factor 0.875 для tablet, 0.75 для mobile)
4. Генерирует preview во всех разрешениях для быстрой проверки

**Конфигурация правил (JSON):**

```json
{
  "breakpoints": {
    "desktop": 1440,
    "tablet": 768,
    "mobile": 375
  },
  "rules": [
    {
      "element": "hero-section",
      "tablet": {
        "layout": "column",
        "padding": "24px",
        "image_position": "top"
      },
      "mobile": {
        "layout": "column",
        "padding": "16px",
        "hide": ["decoration-circle-1", "decoration-circle-2"],
        "font_scale": 0.75
      }
    },
    {
      "element": "features-grid",
      "tablet": {
        "columns": 2,
        "gap": "16px"
      },
      "mobile": {
        "columns": 1,
        "gap": "12px"
      }
    }
  ]
}
```

**Дополнительно:** Python-скрипт для batch-экспорта preview всех брейкпоинтов в PNG для презентации клиенту.

**Инструменты:** Figma Plugin API, Python (Pillow для обработки изображений), Figma REST API.

## Выводы и рекомендации

Автоматизация через AI — это не про замену дизайнеров, а про **освобождение мозга** от рутинных операций. Ключевые принципы, которые я применил:

1. **Автоматизируй только повторяющиеся задачи.** Креатив и стратегию оставь людям.
2. **Начинай с no-code.** n8n и Make позволяют собрать MVP автоматизации за день, без разработчика.
3. **Инвестируй в промпт-инжиниринг.** Качество выхода AI напрямую зависит от качества входа.
4. **Делай fallback на человека.** AI-контент всегда проходит редактуру: дизайнер проверяет смысл, копирайтер — стиль.
5. **Документируй.** Каждый сценарий автоматизации задокументирован: если сломается, любой член команды поймёт логику.

> Самое ценное — это не сэкономленные часы, а **внимание команды**, которое теперь тратится на задачи, где нужен человеческий мозг.
